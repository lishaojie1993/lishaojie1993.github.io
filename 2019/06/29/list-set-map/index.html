集合类不安全问题

ArrayList

ArrayList初始化

public class ListDemo {
    public static void main(String[] args){
        //根据源码得知ArrayList初始化：构造初始容量为10的空数组
        List<String> list = new ArrayList<>();
        list.add("1");
        list.add("2");
        list.add("3");
        for (int i=0;i<3;i++) {
            System.out.println(list.get(i));
        }
    }
}

ArrayList线程不安全

故障现象

java.util.ConcurrentModificationException：并发修改异常

public class ListDemo {
    public static void main(String[] args){
        final List<String> list = new ArrayList<>();
        for (int i=0;i<30;i++) {
            new Thread(new Runnable() {
                public void run() {
                    list.add(UUID.randomUUID().toString().substring(0,8));
                    System.out.println(list);
                }
            }).start();
        }
    }
}
//输出异常
//java.util.ConcurrentModificationException
//    at java.util.ArrayList$Itr.checkForComodification(ArrayList.java:909)
//    at java.util.ArrayList$Itr.next(ArrayList.java:859)
//    at java.util.AbstractCollection.toString(AbstractCollection.java:461)
//    at java.lang.String.valueOf(String.java:2994)
//    at java.io.PrintStream.println(PrintStream.java:821)
//    at ListDemo$1.run(ListDemo.java:13)
//    at java.lang.Thread.run(Thread.java:748)

分析原因

ArrayList线程不安全的原因是因为ArrayList的add()方法的源码中没有加锁，并发争抢修改导致异常。

//源码为了保证并发性和效率，没有加synchronized
public boolean add(E e) {
  ensureCapacityInternal(size + 1);
  elementData[size++] = e;
  return true;
}

解决方案

  1. List list = new Vector<>();
  2. List list = Collections.synchronizedList(new ArrayList<>());
  3. List list = new CopyOnWriteArrayList<>();

CopyOnWrite容器即“写时复制”的容器object[] element，往这个容器里添加元素的时候，不直接往当前容器中添加，而是先拷贝当前容器，在拷贝出的新容器中object[] newElement添加元素，添加完成后再将原容器的引用指向新容器setArray(newElement)，这样的好处是可以对CopyOnWrite容器进行并发的读，是一种读写分离的思想。

HashSet

HashSet的底层实现是HashMap，默认初始容量是16，负载因子是0.75。

虽然底层是HashMap，不过HashSet只关心key，value都是一个PRESENT的常量。

HashSet线程不安全

故障现象

java.util.ConcurrentModificationException：并发修改异常

public class SetDemo {
    public static void main(String[] args){
        final Set<String> set = new HashSet<>();
        for (int i=0;i<30;i++) {
            new Thread(new Runnable() {
                public void run() {
                    set.add(UUID.randomUUID().toString().substring(0,8));
                    System.out.println(set);
                }
            }).start();
        }
    }
}
//java.util.ConcurrentModificationException
//    at java.util.HashMap$HashIterator.nextNode(HashMap.java:1442)
//    at java.util.HashMap$KeyIterator.next(HashMap.java:1466)
//    at java.util.AbstractCollection.toString(AbstractCollection.java:461)
//    at java.lang.String.valueOf(String.java:2994)
//    at java.io.PrintStream.println(PrintStream.java:821)
//    at ListDemo$1.run(SetDemo.java:11)
//    at java.lang.Thread.run(Thread.java:748)

分析原因

add()方法没有加锁。

//PRESENT是一个Object类型的常量
public boolean add (E e) {
  return map.put(e,PRESENT) == null;
}

解决方案

  1. Set set = Collections.synchronizedSet(new HashSet<>());
  2. Set set = new CopyOnWriteArraySet<>();
private final CopyOnWriteArrayList<E> al;
/**
 * Creates an empty set
**/
public CopyOnWriteArraySet() {
  al = new CopyOnWriteArrayList<E>();
}

通过源码可以看出CopyOnWriteArraySet调用的是CopyOnWriteArrayList.

HashMap

故障现象和原因都同上，解决方案使用ConCurrentHashMap。
